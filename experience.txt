AI 도구 실제 업무 적용 경험 — Claude Code로 채용 공고 수집 자동화

================================================================================

■ 배경 및 문제 인식

취업 준비 중 사람인에서 오는 맞춤 공고 추천 메일이 매일 수신됩니다. 그러나 메일을 열어
공고를 하나씩 확인하고, 관심 있는 공고를 별도로 정리하는 과정이 반복적이고 비효율적이었습니다.
메일을 놓치거나, 본 공고를 또 보거나, 정리 자체를 미루는 일이 잦았습니다.

목표: Gmail로 수신되는 사람인 추천 공고 메일을 자동으로 파싱해
      Notion 데이터베이스에 정리되도록 파이프라인을 구축한다.

================================================================================

■ Claude Code 활용 방식

이 프로젝트는 코드를 직접 한 줄씩 작성한 것이 아니라,
Claude Code(Anthropic의 AI 코딩 CLI 도구)를 터미널에서 직접 대화하며 구현했습니다.

구체적인 활용 흐름은 다음과 같습니다.

--------------------------------------------------------------------------------

[1단계 — 구조 설계 요청]

"Gmail에서 사람인 메일을 읽어 Notion에 자동으로 업로드하고 싶다"는 요구사항을
Claude Code에 설명하자, 다음과 같은 모듈 분리 구조를 제안받고 즉시 파일 단위로 생성했습니다.

  Gmail(IMAP) → 메일 수집 → HTML 파싱(공고 추출) → Notion API 업로드

  - gmail_reader.py    : Gmail IMAP 연결 및 사람인 발신 메일 필터링
  - saramin_parser.py  : 메일 HTML에서 공고 제목·회사명·링크 추출
  - notion_uploader.py : Notion API로 데이터베이스 행 추가
  - main.py            : 전체 파이프라인 실행 + 중복 방지

--------------------------------------------------------------------------------

[2단계 — 실제 문제 해결 과정]

코드 생성만으로 끝나지 않았습니다. 사람인 메일의 HTML 구조가 일반적이지 않아
공고 링크 파싱이 처음에는 0개를 반환했습니다. 이 상황을 Claude Code에 전달하자:

  - --debug 플래그를 추가해 실제 메일 HTML을 debug_email.html로 덤프하는 기능을 제안
  - 덤프된 HTML을 함께 분석해 saramin_parser.py의 링크 필터링 로직을
    실제 구조에 맞게 수정

이처럼 "코드 생성 → 실행 → 오류 피드백 → 수정" 사이클을
Claude Code와의 대화로 빠르게 반복했습니다.

--------------------------------------------------------------------------------

[3단계 — 중복 방지 및 자동화 설정]

같은 공고가 반복 업로드되는 문제를 언급하자, seen_jobs.json으로 이미 처리한
공고의 URL/ID를 추적하는 방식을 제안하고 load_seen() / save_seen() 함수를 작성했습니다.

macOS cron 등록 방법도 README에 바로 포함시켜,
매시간 자동 실행 환경까지 함께 완성했습니다.

================================================================================

■ 결과

  사용 도구   | Claude Code (CLI), Python, Gmail IMAP, Notion API
  소요 시간   | 약 1~2시간 (설계~실행 테스트 포함)
  자동화 범위 | 메일 수신 → 공고 파싱 → Notion 업로드 → 중복 제거 전 과정
  외부 비용   | 전부 무료 도구만 사용 (Notion Free, Gmail IMAP, Claude Code)

메일을 수동으로 확인하던 반복 작업이 사라지고,
Notion 데이터베이스에 공고가 자동으로 쌓이는 구조가 완성되었습니다.

================================================================================

■ 핵심 인사이트

  "AI 코딩 도구의 가치는 코드 생성 속도가 아니라,
   실제 오류 상황에서 함께 디버깅하고 구조를 개선하는 과정에 있었다."

Claude Code를 사용하며 얻은 가장 큰 경험은, 막히는 지점(파싱 실패, API 오류, 중복 처리)마다
문제를 그대로 설명하면 원인 분석과 수정 방향을 함께 제시받을 수 있다는 점입니다.
기존에 검색과 StackOverflow를 반복하던 흐름보다 훨씬 빠르게
실동작하는 코드에 도달할 수 있었습니다.
