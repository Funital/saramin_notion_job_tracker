AI 도구 실제 업무 적용 경험 — Claude Code로 채용 공고 수집 자동화

================================================================================

■ 배경 및 문제 인식

취업 준비 중 사람인에서 오는 맞춤 공고 추천 메일이 매일 수신됩니다. 그러나 메일을 열어
공고를 하나씩 확인하고, 관심 있는 공고를 별도로 정리하는 과정이 반복적이고 비효율적이었습니다.
메일을 놓치거나, 본 공고를 또 보거나, 정리 자체를 미루는 일이 잦았습니다.

목표: Gmail로 수신되는 사람인 추천 공고 메일을 자동으로 파싱해 Notion 데이터베이스에
      정리되고, Mac이 꺼져 있어도 매일 자동으로 실행되는 완전한 파이프라인을 구축한다.

================================================================================

■ Claude Code 활용 방식

이 프로젝트는 코드를 직접 한 줄씩 작성한 것이 아니라,
Claude Code(Anthropic의 AI 코딩 CLI 도구)를 터미널에서 직접 대화하며 구현했습니다.

구체적인 활용 흐름은 다음과 같습니다.

--------------------------------------------------------------------------------

[1단계 — 구조 설계 및 코드 생성]

"Gmail에서 사람인 메일을 읽어 Notion에 자동으로 업로드하고 싶다"는 요구사항을
Claude Code에 설명하자, 다음과 같은 모듈 분리 구조를 제안받고 즉시 파일 단위로 생성했습니다.

  Gmail(IMAP) → 메일 수집 → HTML 파싱(공고 추출) → Notion API 업로드

  - gmail_reader.py    : Gmail IMAP 연결 및 사람인 발신 메일 필터링
  - saramin_parser.py  : 메일 HTML에서 공고 제목·회사명·링크 추출
  - notion_uploader.py : Notion API로 데이터베이스 행 추가
  - main.py            : 전체 파이프라인 실행 + 중복 방지

--------------------------------------------------------------------------------

[2단계 — 파싱 오류 디버깅]

코드 생성만으로 끝나지 않았습니다. 사람인 메일의 HTML 구조가 일반적이지 않아
공고 링크 파싱이 처음에는 0개를 반환했습니다. 이 상황을 Claude Code에 전달하자:

  - --debug 플래그를 추가해 실제 메일 HTML을 debug_email.html로 덤프하는 기능을 제안
  - 덤프된 HTML을 함께 분석해 saramin_parser.py의 링크 필터링 로직을
    실제 구조에 맞게 수정

이처럼 "코드 생성 → 실행 → 오류 피드백 → 수정" 사이클을
Claude Code와의 대화로 빠르게 반복했습니다.

--------------------------------------------------------------------------------

[3단계 — 중복 방지 로직 구현]

같은 공고가 반복 업로드되는 문제를 언급하자, seen_jobs.json으로 이미 처리한
공고의 URL/ID를 추적하는 방식을 제안하고 load_seen() / save_seen() 함수를 작성했습니다.

--------------------------------------------------------------------------------

[4단계 — GitHub Actions로 클라우드 자동화 배포]

로컬 스크립트를 Mac이 꺼져도 실행되도록 GitHub Actions로 전환했습니다.
Claude Code가 워크플로 파일(.github/workflows/saramin_to_notion.yml)을 생성했으며,
민감 정보(Gmail 계정, Notion 토큰)는 GitHub Secrets에 안전하게 등록했습니다.

  - 매일 오전 9시 KST(cron: '0 0 * * *') 자동 실행
  - GitHub UI에서 수동 실행(workflow_dispatch)도 가능
  - 실행 후 seen_jobs.json을 repo에 커밋해 중복 이력을 영구 저장

--------------------------------------------------------------------------------

[5단계 — 권한 오류 디버깅 및 수정]

첫 실행에서 seen_jobs.json 커밋 단계가 실패했습니다.
원인은 GitHub Actions 기본 토큰에 쓰기 권한이 없었던 것이었습니다.
Claude Code가 즉시 원인을 파악하고 워크플로에 아래 두 가지를 추가해 해결했습니다.

  - permissions: contents: write  (job 레벨 쓰기 권한 부여)
  - token: ${{ secrets.GITHUB_TOKEN }}  (checkout 시 쓰기 가능한 토큰 사용)

수정 후 재실행에서 전 단계 성공 확인.

================================================================================

■ 최종 결과

  사용 도구   | Claude Code (CLI), Python, Gmail IMAP, Notion API, GitHub Actions
  자동화 범위 | 메일 수신 → 공고 파싱 → Notion 업로드 → 중복 제거 → 클라우드 배포 전 과정
  실행 조건   | Mac 전원과 무관하게 매일 자동 실행 (GitHub 클라우드)
  외부 비용   | 전부 무료 도구만 사용 (Notion Free, Gmail IMAP, GitHub Actions Free tier)

메일을 수동으로 확인하던 반복 작업이 사라지고,
Notion 데이터베이스에 공고가 매일 자동으로 쌓이는 구조가 완성되었습니다.

================================================================================

■ 핵심 인사이트

  "AI 코딩 도구의 가치는 코드 생성 속도가 아니라,
   실제 오류 상황에서 원인을 함께 파악하고 즉시 수정하는 과정에 있었다."

이번 프로젝트에서 Claude Code는 단순한 코드 생성기가 아니었습니다.
파싱 실패, API 오류, GitHub Actions 권한 문제 등 실제로 막히는 지점마다
상황을 설명하면 원인 분석과 수정 방향을 즉시 제시했고,
수정된 코드를 직접 파일에 반영하고 커밋·push까지 수행했습니다.

기획 → 구현 → 디버깅 → 배포의 전 과정을 하나의 대화 흐름 안에서 완결할 수 있었으며,
이는 기존에 검색과 문서를 반복하던 방식 대비 체감상 수 배 이상 빠른 속도였습니다.
